Сегментація клієнтів E-commerce (RFM-аналіз) 📊

📌 Про проєкт

Цей проєкт присвячений аналізу купівельної поведінки клієнтів за допомогою моделі RFM (Recency, Frequency, Monetary). Головна мета — сегментувати користувачів на основі історії їхніх замовлень. Це допомагає маркетинговій команді ефективніше налаштовувати кампанії, підвищувати рівень утримання клієнтів (retention) та виявляти найбільш цінних покупців.

🛠️ Стек технологій

Python (pandas, numpy): Генерація тестових даних (симуляція реальних сирих логів інтернет-магазину).

SQL (Google BigQuery): Трансформація даних, розвідувальний аналіз (EDA) та розрахунок RFM-скорингу за допомогою CTE (Common Table Expressions) та віконних функцій.

Tableau Public: Візуалізація даних та побудова інтерактивного дашборду.

📂 Процес та методологія

Генерація даних: Написано Python-скрипт для створення датасету з 5 000 користувачів та 12 000 транзакцій за 18-місячний період.

Налаштування сховища (DWH): Сирі .csv файли завантажено у хмарне сховище Google BigQuery.

Трансформація даних (SQL):

Розраховано базові метрики R, F та M для кожного користувача.

Використано віконну функцію NTILE(5) для присвоєння оцінок від 1 до 5 за кожним критерієм.

Застосовано логіку CASE WHEN для групування оцінок у 7 бізнес-сегментів (наприклад, "Чемпіони", "У зоні ризику", "Новачки").

Візуалізація даних: Фінальну вітрину даних експортовано в Tableau для побудови візуального розподілу клієнтської бази.
